# 補課系統移交手冊

> 適用：將此系統獨立部署給另一分院使用（完全獨立部署——各分院各自 Fork 一份程式碼＋
> 各自的 Supabase／Vercel，資料完全分開，互不影響）
> 前提：貴分院已有中台 zenclass 出缺勤系統的帳號與**單位代碼（UNIT_ID）**
> （zenclass 是中台總部的刷卡出缺勤系統，本系統會自動去抓 zenclass 的資料）
> 預計操作時間：約 40–50 分鐘（不含之後的自動同步設定）
> 所需準備：電腦、瀏覽器、可收信的 email 帳號

---

## 事前準備

開始操作前，請準備以下資訊：

- 師父的 email（用於登入後台）
- 自訂一組後台密碼（至少 6 碼）
- 貴分院的 zenclass 單位代碼（**UNIT_ID**，例如普宜精舍是 `UNIT01071`）——這是中台編給
  各分院的代碼，向總窗口／中台詢問即可取得

本系統不需要安裝任何程式即可上線；後續若要接上「出缺勤自動同步」，需要另外一台現場電腦
安裝一支小工具（見「第十步」），但這不影響系統本身能不能先上線使用。

---

## 第一步：申請 Supabase 帳號（資料庫）

Supabase 為存放學員名冊、出缺勤、補課紀錄的資料庫服務，使用免費方案即可。

1. 開啟瀏覽器，前往 **https://supabase.com**
2. 點選右上角 **「Start your project」** 或 **「Sign Up」**
3. 選擇以 GitHub 帳號登入，或以 email＋密碼註冊
4. 登入後，點選 **「New Project」**
5. 填入以下資訊：
   - **Name**：可自訂，例如 `xinyi-buke`（僅限英文、數字、連字號）
   - **Database Password**：自訂一組，**請妥善保存**
   - **Region**：選擇 **Northeast Asia (Tokyo)**，離台灣最近
6. 點選 **「Create new project」**，約 1 分鐘後建置完成

---

## 第二步：建立資料庫

1. 進入剛建立的 Supabase 專案
2. 於左側選單點選 **「SQL Editor」**
3. 點選右上角 **「New query」**
4. 開啟新分頁，前往程式碼庫：
   **https://github.com/nirvana0720/puyi-buke**
5. 找到並開啟 **`db/full_setup_all_in_one.sql`**
   （此檔案已將全新環境建置所需的所有資料表與函式依正確順序合併為單一檔案，
   僅需執行這一份，不需另外找其他 SQL 檔案）
6. 點選右上角 **「Raw」**，全選內容（Ctrl+A）並複製（Ctrl+C）
7. 回到 Supabase SQL Editor，貼上（Ctrl+V）
8. 點選右下角 **「Run」** 執行
9. 檔案內容較大，執行約需 10 至 30 秒，請耐心等候，勿中途關閉分頁

執行完成後，請依以下步驟確認：

- 於左側選單點選 **「Table Editor」**
- 應可看到 17 張左右的資料表（如 members、classes、sessions、makeups、transfers、
  staff_accounts 等）
- 確認無誤即代表建置成功

> ⚠️ **本合併檔案僅適用於「全新環境、首次建置」。** 若日後貴分院要更新到系統的最新版本
> （例如新增了某個功能），**請勿重新執行整份合併檔案**——目前這套系統還沒有像
> 「單支檔案自動比對版本」這樣的工具，資料庫異動會由普宜精舍主動通知並提供對應的
> SQL，詳見「第十一步：日後如何取得系統更新」。
>
> ⚠️ 本檔案執行後，尾端註解有列出「這支 SQL 做不到、還要另外做的事」（例如金鑰要換、
> 管理員帳號要另外建），下面第三步起就是依序把這些事情做完。

---

## 第三步：取得金鑰

以下兩組金鑰用於連接前台網站與資料庫。

1. 於 Supabase 左側選單點選 **「Project Settings」**（齒輪圖示）
2. 點選 **「API」**
3. 取得以下兩項數值並分別複製保存：
   - **Project URL**：格式類似 `https://xxxxxxxx.supabase.co`
   - **anon public**（位於 Project API Keys 區塊，新版介面可能顯示為
     「Publishable key」）：一組字串

> ⚠️ anon / publishable key 為公開金鑰，可用於前端程式碼。**請勿複製 service_role key**。

## 第四步：Fork 程式碼＋設定連線資訊

本系統**沒有使用 Vercel 的環境變數**——分院名稱、Supabase 連線資訊等設定，全部直接寫在
程式碼裡的 `config/config.js` 這一支檔案，換分院部署就是改這支檔案，不是填 Vercel 的
Environment Variables。

1. 開啟瀏覽器，前往 **https://github.com/nirvana0720/puyi-buke**
2. 點選右上角 **「Fork」**
3. 選擇自己的帳號，其餘設定維持預設，點選 **「Create fork」**
4. 完成後網址將變為 `https://github.com/您的帳號/puyi-buke`，此即為專屬副本，後續部署與
   之後取得更新皆以此為準

> ⚠️ **這一步非常重要**：Fork 完成的當下，`config/config.js` 裡裝的還是**普宜精舍真實的
> Supabase 金鑰**（這支檔案本來就有進 git，方便原分院自己維護）。**部署上線前一定要把它
> 換成貴分院自己第三步取得的金鑰**，否則貴分院的網站會連到普宜精舍的資料庫，兩邊資料
> 會混在一起。

5. 在自己 Fork 的頁面，點開 **`config`** 資料夾 → 點選 **`config.js`**
6. 點選右上角鉛筆圖示（Edit this file）
7. 依下表把內容改成貴分院自己的值（其餘欄位如 `LATE`、`MAKEUP`、`MARK`、`GRADUATION`
   維持不動即可，那些是規則設定，不是連線資訊）：

   | 欄位 | 改成 |
   |------|----|
   | `TEMPLE_NAME` | 貴分院名稱，例如 `'妙開精舍'`（各頁標題會自動帶入） |
   | `SUPABASE_URL` | 第三步複製的 Project URL |
   | `SUPABASE_ANON_KEY` | 第三步複製的 anon public（publishable）key |
   | `UNIT_ID` | 貴分院的 zenclass 單位代碼 |

8. 拉到頁面下方，點選 **「Commit changes...」→「Commit directly to the main branch」**

## 第五步：部署到 Vercel

Vercel 為網站的上線平台，使用免費方案即可。這個系統是純靜態網頁（沒有 Vite／React 這類
建置流程），部署設定比一般網站簡單，不需要填任何環境變數。

1. 前往 **https://vercel.com**
2. 點選 **「Sign Up」**，建議以 **GitHub** 帳號登入
3. 登入後，點選 **「Add New Project」**
4. 選擇 **「Import Git Repository」**
5. 搜尋或輸入**自己的** `您的帳號/puyi-buke`（並非 `nirvana0720/puyi-buke`）
6. 點選 **「Import」**
7. **Framework Preset** 選擇 **「Other」**（本站沒有建置指令，Build/Output 設定維持預設即可）
8. 不需要填寫任何 Environment Variables（第四步已經把設定直接寫進程式碼裡了）
9. 點選 **「Deploy」**
10. 約 1 分鐘後，畫面顯示 **「Congratulations!」** 即代表部署成功
11. 請記錄網站網址，格式類似 `https://puyi-buke-xxxx.vercel.app`

> 📌 **為何要用自己的 Fork，不能直接匯入 `nirvana0720/puyi-buke`**：原始程式碼庫貴分院
> 並無寫入權限，日後系統更新也無法自動同步過來。用自己的 Fork，之後只需點一次
> 「Sync fork」即可自動重新部署，詳見「第十一步」。
>
> 📌 日後可於 Vercel 的 Project Settings → Domains 設定自訂網域（非必要）

---

## 第六步：建立管理員帳號（後台登入）

「精舍後台」（`/ui/admin/`）用的是 **Supabase Auth**（email＋密碼）登入，跟下一步的
「義工帳號」是兩套不同的系統，不要搞混。

1. 回到 **Supabase** 後台
2. 於左側選單點選 **「Authentication」**
3. 點選 **「Users」** → 右上角 **「Add user」** → **「Create new user」**
4. 填入師父的 email 與密碼
5. 勾選 **「Auto Confirm User」**（無需驗證信）
6. 點選 **「Create User」**

完成即可，**不需要**再另外跑 SQL 設定角色/權限——本系統的後台登入判斷比較單純，只要是
一組有效的 Supabase Auth 帳密，登入後就能看到完整的後台管理功能。

---

## 第七步：測試登入後台

1. 開啟第五步取得的網址，例如 `https://puyi-buke-xxxx.vercel.app/ui/admin/`
2. 輸入師父的 email 與密碼，點選登入
3. 成功進入後台（左側會出現「班別」「學員」「義工帳號」等選單），即代表部署完成

## 第八步：建立義工帳號（義工櫃台用）

義工在現場「義工櫃台」（`/ui/kiosk/`）登記補課、點名報到，用的是**帳號＋密碼**（不是
email），跟後台是分開的兩套帳號系統。

1. 登入後台，點選左側 **「👷 義工帳號」**
2. 點選 **「新增義工帳號」**，填入帳號名稱、密碼（至少 6 碼）、顯示名稱
3. 角色可選 **volunteer**（一般義工，只能進櫃台）或 **admin**（也能登入後台）
4. 完成後，義工即可用這組帳密登入 `/ui/kiosk/`

---

## 第九步：建立班別、匯入學員名冊

1. 於後台點選左側 **「班別」→「新增班別」**，填入班名、星期、日夜間、堂數等資料
2. 點選左側 **「匯入資料」**，上傳學員名冊 Excel（欄位會自動依表頭關鍵字對應，例如
   「學員編號」「姓名」「法名」「組別」「執事」等，不需要完全照固定欄位順序）
3. 詳細操作方式（怎麼設定補課規則、怎麼指派學長班長等）請另外跟普宜精舍索取
   《補課系統操作手冊》——這份日常操作說明因為附有實際畫面截圖，沒有放進公開的程式碼庫，
   會由普宜精舍另外傳送（email 或雲端連結）；本手冊只涵蓋「從無到有部署一個新環境」這件事

> 📌 **關於 zenclass 綁定**：後台建立班別後，若尚未綁定 zenclass 的真實班級代碼，
> 登入後台會看到紅色提示「有 N 班尚未綁定 zenclass 真代碼」，點進去依指示綁定即可，
> 綁定後出缺勤自動同步（下一步）才抓得到這個班的資料。

## 第十步：接上出缺勤自動同步（zenclass）

這是本系統的核心功能——把 zenclass 刷卡出缺勤自動同步進補課系統，不用義工手動登打。
共有兩種方式，**建議都設定**（一個自動、一個手動備援，互不衝突）：

### 10.1 自動同步（node_syncer，主要方式）

讓精舍的一台電腦，每天固定 12 個時段自動同步，取代人工點書籤。詳細步驟已經寫成一份
完整說明：**`grabber/node_syncer/README.txt`**（這支檔案跟著程式碼一起進 Fork，一定
拿得到；另有一份內容相同的完整版 SOP 留在原分院本機參考用，未必會在您 Fork 的副本裡），
照著做即可，這裡列出跟「完全獨立部署」情境唯一不同的一點：

> ⚠️ **這份說明原本是寫給「跟普宜共用同一個 Supabase」的分院看的**，裡面步驟 2 說
> 「`SUPABASE_URL`／金鑰通常不用動，大家共用同一套補課系統」——**這句話不適用於貴分院**。
> 貴分院是完全獨立的 Supabase 專案，`grabber/node_syncer/config.json` 裡的
> `SUPABASE_URL`、`SUPABASE_ANON_KEY`、`UNIT_ID` **三項都要改成貴分院自己的值**
> （照 `config.example.json` 的格式），不能只改 `UNIT_ID`。其餘安裝步驟（下載
> `node.exe`、複製到現場電腦、`test_sync_now.bat` 測試、`setup_schedule.bat` 建排程）
> 完全比照 README.txt 操作即可。

### 10.2 手動備援（瀏覽器書籤）

排程漏跑，或要手動補抓過去某一天的資料時，可以用瀏覽器書籤手動同步。**重新產生書籤前，
一定要先換成貴分院自己的金鑰**，步驟：

1. 在自己電腦上把 Fork 的程式碼複製一份下來（`git clone` 自己的 repo，或直接下載 zip）
2. 確認 `config/config.js` 已經是第四步改好、貴分院自己的設定值
3. 開啟終端機，切到程式碼資料夾，執行：
   ```
   node grabber/bookmarklet_build.js
   node grabber/bookmarklet_build.js bookmarklet_quick.js
   node grabber/bookmarklet_build.js audit_bookmarklet.js
   ```
4. 每一行執行完，畫面會印出一長串 `javascript:...` 開頭的文字，複製它，存成瀏覽器書籤的
   「網址」欄位（三支功能分別是：一般同步、快速同步、稽核比對）

> ⚠️ **這一步千萬不能省略。** 如果直接使用 Fork 下來、還沒重新產生過的書籤（`.min.txt`
> 檔案本身也已 build 好存在 git 裡），書籤裡包的是**普宜精舍真實的金鑰**，點下去會把
> 資料同步到普宜精舍的資料庫，不是貴分院自己的。

## 第十一步：日後如何取得系統更新

由於第四步已採用專屬的程式碼副本進行部署，日後系統更新時無需重新設定 Vercel，僅需：

1. 開啟自己 Fork 的頁面 `https://github.com/您的帳號/puyi-buke`，若有新版本，畫面將顯示
   **「This branch is out-of-date」** 等提示，點選 **「Sync fork」→「Update branch」**
2. 因為 Vercel 已連結至自己的程式碼副本，同步完成後將自動偵測新的 commit 並重新部署，
   約 1 至 2 分鐘後可重新整理確認

> ⚠️ **此操作僅同步「程式碼」，不會自動同步「資料庫」。** 這套系統目前還沒有像
> 「一份文件列出所有版本紀錄、一支腳本自動檢查資料庫是否跟上」這樣的工具，
> 若某次更新涉及資料庫異動（新增欄位、新增函式），**普宜精舍會主動聯絡貴分院**，
> 提供對應的 SQL 與說明，貴分院不用自己判斷要不要跑。一般性的文字調整或不影響資料庫
> 的小幅修正，可能不會另行通知。
>
> 貴分院自行修改的檔案（例如 `config/config.js`）不會被同步操作覆蓋——「Sync fork」
> 只會拉入原始程式碼有異動的部分，貴分院自己改過的設定會保留。

## 常見問題

**Q：部署到 Vercel 後，開啟網站是空白或找不到頁面？**
確認 Framework Preset 是否選了「Other」，以及 Import 的是自己 Fork 的
`您的帳號/puyi-buke`，而不是原始的 `nirvana0720/puyi-buke`。

**Q：登入後台顯示「登入失敗」？**
請確認第六步是否已在 Supabase Authentication → Users 建立帳號，且勾選了
「Auto Confirm User」（否則帳號會停在未驗證狀態，無法登入）。

**Q：義工在櫃台輸入帳密登入不了？**
義工帳號跟後台帳號是分開的兩套系統，義工帳號要在後台「義工帳號」面板建立（第八步），
不是在 Supabase Authentication 建立。

**Q：學員掃 QR code／輸入編號登入沒有反應？**
請確認學員名冊已經匯入（第九步），且該學員的編號跟 QR code 內容一致（9 碼數字）。

**Q：出缺勤同步不到、或同步到別班的資料？**
最常見原因是 `config.json`（node_syncer）裡的 `UNIT_ID` 填錯，改成貴分院正確的
zenclass 單位代碼即可。

**Q：忘記 Supabase 密碼，該如何處理？**
Supabase 網站提供「Forgot password」重設功能。

**Q：如何更換精舍名稱？**
改 `config/config.js` 的 `TEMPLE_NAME` 即可，全站標題會自動帶入，不需要改 Vercel 設定。

## 需要普宜精舍協助處理之事項

以下事項仍需請普宜精舍（或熟悉此系統之人員）協助處理：

- **系統功能開發**：程式錯誤修復或新功能開發，仍須由普宜精舍端進行，因各分院係各自獨立
  的 Supabase＋Vercel 環境，**任一分院之修改不會自動同步至其他分院**
- **資料庫異動的通知**：涉及資料表、函式異動的更新，普宜精舍會主動聯絡並提供對應 SQL

以下事項可由分院自行處理，無需另尋協助：

- **取得程式碼更新**：於自己 Fork 的 GitHub 頁面點選「Sync fork」，Vercel 即自動重新
  部署，詳見「第十一步」
- **出缺勤自動同步安裝**：照 `grabber/node_syncer/README.txt` 步驟自行安裝
  （注意第十步 10.1 提到的金鑰差異）
- **建立義工帳號、班別、匯入名冊**：後台介面皆可自行操作，詳見操作手冊
