補課系統 — 現場排程同步器（node_syncer） 部署說明
====================================================

【這是什麼】
讓精舍的一台電腦，每天固定 12 個時段，自動把 zenclass 刷卡出缺勤同步進補課系統，
取代義工手動點書籤。它自帶 Node 引擎（含新版 TLS），不管電腦多舊——連 Windows 7
都能跑，不用裝框架、不用改系統、不用管理員。

【要準備什麼】
一台「有網路」的電腦（例如師父自己的電腦）先做一次下載；之後把整個資料夾
複製到「要負責跑同步、平常會開著」的那台電腦即可。

────────────────────────────────────────
步驟 1：下載引擎（在「有網路」的電腦上做一次）
────────────────────────────────────────
雙擊「1_download_node.bat」。
會自動把官方 node.exe 下載進這個資料夾。
（若失敗：用瀏覽器打開下面網址，下載後把 node.exe 放進這個資料夾）
  https://nodejs.org/dist/v12.22.12/win-x86/node.exe

做完，這個資料夾裡應該要看得到 node.exe。

────────────────────────────────────────
步驟 2：確認設定
────────────────────────────────────────
用記事本打開 config.json，確認：
  UNIT_ID  = 這個精舍的單位代碼（普宜精舍是 UNIT01071；別的精舍要改成自己的）
其餘（金鑰、網址）通常不用動。
※ 要部署到別的精舍時，照 config.example.json 的格式改 config.json 即可。

────────────────────────────────────────
步驟 3：部署到「要跑同步」的那台電腦
────────────────────────────────────────
1. 把「整個資料夾」（含 node.exe）複製過去（USB 或區網都行）。
2. 先手動測一次：雙擊「test_sync_now.bat」。
   黑視窗若出現「===== 開始同步 ... =====」和各班「OK ...同步成功」＝成功。
3. 測試 OK 後，雙擊「setup_schedule.bat」→ 一次建好 12 個自動排程。完成！

【同步時段（台北時間）】
早 09:20 / 09:25 / 09:30 / 10:00 / 10:10 / 11:00
晚 19:20 / 19:25 / 19:30 / 20:00 / 20:10 / 21:00

【疑難排解】
- 雙擊 test 沒反應或報缺少 DLL：該 Win7 可能要先裝 Universal C Runtime（KB2999226）。
- 想看歷史紀錄：本資料夾的 sync_log.txt 有每次執行的完整結果。
- 換精舍 / 換電腦：整個資料夾複製過去，改 config.json 的 UNIT_ID，重跑 setup_schedule.bat。
- 手動補抓過去日期：仍可用原本的書籤（bookmarklet），本工具不影響書籤。
